# Двадцать одно (Blackjack 21)

Готовая к публикации HTML5-игра «21» (Blackjack) для **Яндекс Игр**.
Полностью работает в браузере, без сервера, офлайн, на телефонах, планшетах и ПК.

![Стиль](в стиле карточных игр Яндекса — зелёный стол, оранжевый риббон)

---

## ✨ Что внутри

- **Классические правила блэкджека**: раздача по 2 карты, Hit / Stand / Double,
  дилер добирает до 17+, туз = 1 или 11 автоматически, перебор (bust),
  натуральный блэкджек с выплатой 3:2, ничья (push).
- **Ставки и фишки**: баланс монет, фишки 10/25/50/100/500, выплаты, бонус при
  обнулении баланса — игра никогда не попадает в тупик.
- **UI уровня мини-казино**: зелёный фетровый стол с ромбовидным узором,
  оранжевый риббон-логотип, настоящие карты, покерные фишки.
- **Анимации**: вылет карт из колоды, переворот в 3D, плавный счётчик очков,
  всплывающий баннер результата, свечение руки-победителя, glow/shine на картах.
- **Звук**: синтез на Web Audio API (раздача, клик, фишка, победа, блэкджек,
  проигрыш) — без аудиофайлов, мгновенно.
- **Адаптивность**: mobile-first, корректно масштабируется на планшеты и ПК,
  учитывает «чёлку» (safe-area), уважает `prefers-reduced-motion`.
- **Вибро-отклик** на мобильных (если поддерживается).
- **Сохранение** баланса, статистики и настроек в `localStorage`.
- **Статистика**: раздачи, победы/поражения, блэкджеки, серия, рекорд баланса.
- **Интеграция с Yandex Games SDK** — опциональная и безопасная: вне платформы
  просто отключается, игра продолжает работать.

## 🚀 Как запустить локально

Игра работает по `file://`, но из-за политики браузеров с локальным сервером
надёжнее. Любой из вариантов:

```bash
# Python
python3 -m http.server 8080
# затем откройте http://localhost:8080
```

```bash
# Node (если установлен npx)
npx serve .
```

Либо просто откройте `index.html` двойным кликом.

> Внешний скрипт Yandex SDK подключён в `index.html`. Локально он не загрузится —
> это нормально: код это обрабатывает, и игра работает без SDK.

## 📦 Как опубликовать в Яндекс Играх

1. Заархивируйте **содержимое** этой папки в `.zip` (важно: `index.html` должен
   лежать в корне архива, а не во вложенной папке). Готовый архив —
   `blackjack-21-yandex.zip` рядом с проектом.
2. Зайдите в [Консоль разработчика Яндекс Игр](https://games.yandex.ru/console)
   → создайте черновик → загрузите ZIP.
3. Заполните карточку игры (название «Двадцать одно», категория «Карточные»,
   иконка 512×512, скриншоты).
4. SDK уже подключён и вызывает `LoadingAPI.ready()` после загрузки — Яндекс
   уберёт свой прелоадер автоматически. Межраундовая реклама (`fullscreen`)
   подключена безопасно и показывается изредка.
5. Отправьте на модерацию.

Соответствие требованиям платформы:
- ✅ старт < 3 с (все ассеты ~1.2 МБ, звук синтезируется);
- ✅ не требует сервер и backend;
- ✅ работает в iframe;
- ✅ оптимизировано под слабые устройства (DOM-рендер, без тяжёлых движков);
- ✅ не перегружает GPU/CPU.

## 🧱 Архитектура

Код разделён на слои согласно ТЗ (Game Engine / State Manager / UI Layer /
Asset Manager) и не зависит от внешних библиотек.

```
index.html            точка входа и разметка
style.css             всё оформление (тема Яндекса)
/src
  /core               инфраструктура
    Storage.js          localStorage (баланс, статистика, настройки) + fallback
    AudioManager.js     синтез звуков на Web Audio (аналог Howler.js)
    AssetManager.js     предзагрузка карт + определение путей + fallback
    Haptics.js          вибро-отклик
    YandexSDK.js        безопасная обёртка над Yandex Games SDK
  /game               игровая логика (без DOM)
    Deck.js             колода 52 карты, перемешивание Фишера—Йетса
    Hand.js             подсчёт очков, логика туза 1/11, блэкджек
    BlackjackEngine.js  правила, состояние, ставки, выплаты, статистика
  /ui                 представление
    CardView.js         DOM-карта (3D-переворот + CSS-fallback)
    Anim.js             анимации (Web Animations API, аналог GSAP)
    UI.js               экраны, HUD, оркестрация раунда и тайминга
  main.js             бутстрап: SDK → UI → предзагрузка → готовность
/assets
  /cards              PNG-карты (52 шт. + рубашка), под свободной лицензией
  /sounds             см. README — звук процедурный
```

### Почему такой стек

ТЗ разрешает любой современный стек при условии стабильной работы. Выбран
**чистый JS + CSS без CDN-зависимостей**, потому что для Яндекс Игр это даёт:
- максимально быстрый старт и отсутствие риска блокировки внешних CDN в iframe;
- лёгкость для слабых устройств (DOM-рендер карт + CSS-анимации вместо WebGL);
- мгновенный звук без загрузки файлов.

Роли запрошенных библиотек закрыты нативно: рендер/анимации — Web Animations API
и CSS-трансформации (аналог GSAP/Pixi tweening), звук — Web Audio API
(аналог Howler.js). При желании любой слой легко заменить на Pixi/Phaser/Howler —
логика (`/game`) от представления не зависит.

## 🎮 Управление и правила

- **Ставка**: тапайте по фишкам, чтобы набрать сумму, затем «Раздать».
- **Взять (Hit)** — взять ещё карту.
- **Хватит (Stand)** — остановиться, ход переходит дилеру.
- **Удвоить (×2)** — удвоить ставку и взять ровно одну карту (доступно на первых
  двух картах, если хватает фишек).
- **Следующая раздача** — начать новый раунд (рестарт).
- Цель — набрать сумму ближе к 21, чем у дилера, не превысив 21.
  Очки: 2–10 по номиналу, J/Q/K = 10, туз = 1 или 11. Блэкджек (туз + «десятка»
  с первых двух карт) платит 3:2.

## 🔧 Кастомизация

- Стартовый баланс, минимальная ставка, правило дилера — константы в начале
  `src/game/BlackjackEngine.js`.
- Номиналы фишек — массив `CHIPS` в `src/ui/UI.js`.
- Цвета и размеры — CSS-переменные в `:root` в `style.css`.
- Звуки — объект `SFX` в `src/core/AudioManager.js`.
